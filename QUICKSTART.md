# Быстрый старт OrderManager

## Установка и первый запуск

### 1. Установка зависимостей

```bash
# Запустите батник установки
install.bat
```

Или вручную:
```bash
pip install -r requirements.txt
```

### 2. Инициализация базы данных

```bash
# Запустите батник инициализации
init_db.bat
```

Или вручную:
```bash
python src/main.py --init-db
```

### 3. Запуск приложения

```bash
# Запустите батник
run.bat
```

Или вручную:
```bash
python src/main.py
```

## Первоначальная настройка

### Шаг 1: Добавьте товары

1. Откройте вкладку **Товары**
2. Нажмите **Добавить товар**
3. Заполните поля:
   - Название (обязательно)
   - Категория
   - Цена (обязательно)
   - Единица измерения
   - Штрихкод
   - Загрузите изображение (опционально)
4. Нажмите **Сохранить**

**Или импортируйте из 1С:**
1. Меню **1С (CommerceML)** → **Импорт товаров из 1С**
2. Выберите файлы `import.xml` и `offers.xml`
3. Нажмите **Импортировать**

### Шаг 2: Добавьте менеджеров

1. Откройте вкладку **Менеджеры**
2. Нажмите **Добавить менеджера**
3. Заполните поля:
   - PIN-код (4-10 цифр, уникальный)
   - Имя
   - Телефон
   - Email
4. Нажмите **Сохранить**

### Шаг 3: Настройте WebDAV (для синхронизации)

1. Откройте вкладку **Настройки**
2. Перейдите на подвкладку **WebDAV**
3. Заполните поля:
   - URL сервера: `https://your-server.com/dav/`
   - Логин
   - Пароль
4. Нажмите **Тест подключения**
5. Если тест успешен, нажмите **Сохранить**

### Шаг 4: Настройте бонусную систему

1. Откройте вкладку **Настройки**
2. Перейдите на подвкладку **Бонусная система**
3. Настройте параметры:
   - Процент начисления (например, 5%)
   - Максимум оплаты бонусами (например, 50%)
   - Срок действия бонусов (дни)
4. Нажмите **Сохранить**

## Основные операции

### Просмотр заказов

1. Откройте вкладку **Заказы**
2. Используйте фильтры:
   - Поиск по номеру/клиенту/телефону
   - Статус заказа
   - Статус оплаты
   - Диапазон дат
3. Двойной клик по заказу - детальный просмотр

### Работа с клиентами

1. Откройте вкладку **Клиенты**
2. Поиск по имени или телефону
3. Двойной клик по клиенту - просмотр:
   - Адреса доставки
   - История заказов
   - История бонусов

### Управление товарами

1. Откройте вкладку **Товары**
2. Фильтры по категории и статусу
3. Редактирование: двойной клик по товару
4. Стоп-лист: кнопка в колонке "Активен"

### Синхронизация с устройствами

**Ручная синхронизация:**
1. Нажмите F5 или кнопку **Синхронизировать**
2. Выберите режим:
   - Только выгрузка (отправить данные)
   - Только загрузка (получить данные)
   - Полная синхронизация
3. Нажмите **Начать синхронизацию**

**Автоматическая синхронизация:**
1. Вкладка **Настройки** → **Синхронизация**
2. Включите автоматическую синхронизацию
3. Установите интервал (минуты)
4. Нажмите **Сохранить**

### Аналитика

1. Откройте вкладку **Аналитика**
2. Выберите период:
   - Сегодня
   - Вчера
   - Последние 7/30 дней
   - Этот месяц
   - Произвольный период
3. Просмотрите:
   - Общую статистику
   - График продаж по дням
   - ТОП-10 товаров
   - ТОП-10 клиентов
   - Статистику по менеджерам

### Интеграция с 1С

**Импорт товаров:**
1. Меню **1С (CommerceML)** → **Импорт товаров из 1С** (Ctrl+I)
2. Выберите `import.xml` и `offers.xml`
3. Настройте опции импорта
4. Нажмите **Импортировать**

**Экспорт заказов:**
1. Меню **1С (CommerceML)** → **Экспорт заказов в 1С** (Ctrl+Shift+E)
2. Выберите путь для сохранения `orders.xml`
3. Настройте фильтры (период, статусы)
4. Нажмите **Экспортировать**

## Горячие клавиши

- **F5** - Синхронизация
- **Ctrl+E** - Экспорт данных
- **Ctrl+B** - Резервная копия
- **Ctrl+I** - Импорт из 1С
- **Ctrl+Shift+E** - Экспорт в 1С
- **Ctrl+,** - Настройки
- **Ctrl+Q** - Выход

## Структура данных

### Локальные файлы

```
Central_office/
  data/
    ordermanager.db          # База данных SQLite
    images/                  # Изображения товаров
      products/
        PROD-123.jpg         # Полное изображение (800x800)
        PROD-123_thumb.jpg   # Миниатюра (200x200)
    webdav/                  # Локальные файлы синхронизации
      {PIN}/
        incoming/            # Файлы для устройства
        outgoing/            # Файлы от устройства
        processed/           # Обработанные файлы
```

### WebDAV структура

```
/webdav/
  {PIN_менеджера}/
    incoming/
      products.json          # Список товаров
      settings.json          # Настройки системы
    outgoing/
      orders_*.json          # Заказы от менеджера
      clients_*.json         # Новые клиенты
      payments_*.json        # Координаты оплат
```

## Устранение проблем

### Приложение не запускается

1. Проверьте установку Python 3.8+
2. Установите зависимости: `install.bat`
3. Инициализируйте БД: `init_db.bat`
4. Проверьте логи в консоли

### Ошибка подключения к БД

1. Проверьте наличие файла `data/ordermanager.db`
2. Запустите `init_db.bat`
3. Проверьте права доступа к папке `data/`

### WebDAV не работает

1. Проверьте настройки во вкладке **Настройки → WebDAV**
2. Нажмите **Тест подключения**
3. Проверьте доступность сервера
4. Проверьте логин и пароль

### Синхронизация не работает

1. Проверьте настройки WebDAV
2. Проверьте наличие папок на сервере
3. Проверьте формат JSON файлов
4. Посмотрите лог синхронизации

### Изображения не отображаются

1. Проверьте наличие папки `data/images/products/`
2. Проверьте права доступа
3. Проверьте формат изображений (JPG, PNG)
4. Перезагрузите изображение через редактирование товара

## Рекомендации

1. **Регулярное резервное копирование** - делайте бэкап БД ежедневно
2. **Мониторинг синхронизации** - проверяйте лог синхронизации на ошибки
3. **Обновление товаров** - синхронизируйте справочники перед началом рабочего дня
4. **Проверка данных** - периодически проверяйте корректность заказов и клиентов
5. **Безопасность** - используйте HTTPS для WebDAV, сложные пароли
