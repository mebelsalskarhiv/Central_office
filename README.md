# Центральная система управления OrderManager

Центральная система для управления заказами, клиентами, товарами и менеджерами с синхронизацией через WebDAV.

## Архитектура

### База данных (11 таблиц)

- `managers` - менеджеры (PIN, имя, статус, устройство, последняя синхронизация)
- `clients` - клиенты (телефон, имя, бонусы, статистика)
- `addresses` - адреса клиентов (текст, координаты, метка)
- `products` - товары (название, цена, категория, изображение, штрихкод)
- `orders` - заказы (номер, статус, сумма, оплата, доставка)
- `order_items` - позиции заказов (товар, количество, цена)
- `bonus_transactions` - транзакции бонусов (начисление/списание)
- `payment_locations` - координаты оплат (GPS данные)
- `sync_logs` - лог синхронизации (статус, ошибки, время)
- `settings` - настройки системы (ключ-значение)
- `price_history` - история изменения цен (опционально)

### Синхронизация WebDAV

**Структура папок:**
```
/webdav/
  {PIN_менеджера}/
    incoming/          # ToBase (от центра к устройству)
      products.json    # Список товаров
      settings.json    # Настройки системы
    outgoing/          # FromBase (от устройства к центру)
      orders_*.json    # Заказы
      clients_*.json   # Клиенты
      payments_*.json  # Координаты оплат
    processed/         # Архив обработанных файлов
```

**Процесс синхронизации:**
1. **Выгрузка (ToBase):**
   - Генерация products.json (все активные товары)
   - Генерация settings.json (бонусы, доставка, синхронизация)
   - Загрузка на WebDAV сервер в папку incoming/

2. **Загрузка (FromBase):**
   - Скачивание файлов из папки outgoing/
   - Парсинг и валидация JSON
   - Разрешение конфликтов (Last-Write-Wins по timestamp)
   - Сохранение в БД
   - Перемещение в processed/

3. **Разрешение конфликтов:**
   - Сравнение updated_at timestamps
   - Защита доставленных заказов от перезаписи
   - Баланс бонусов всегда из центральной БД
