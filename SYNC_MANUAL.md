# Руководство по ручной синхронизации

## Описание

Диалог ручной синхронизации позволяет вручную выгружать данные на устройства менеджеров и загружать данные с устройств через WebDAV сервер.

## Запуск

1. **Через меню:** Синхронизация → Синхронизировать сейчас
2. **Через панель инструментов:** Кнопка "Синхронизировать"
3. **Горячая клавиша:** F5

## Режимы синхронизации

### 1. Только выгрузка (отправить данные на устройства)
- Генерирует `products.json` - список всех активных товаров
- Генерирует `settings.json` - настройки системы (бонусы, доставка, синхронизация)
- Загружает файлы на WebDAV сервер в папку `{PIN}/incoming/`
- Используется для обновления справочников на устройствах

### 2. Только загрузка (получить данные с устройств)
- Скачивает файлы из папки `{PIN}/outgoing/` с WebDAV сервера
- Обрабатывает файлы:
  - `orders_*.json` - новые заказы от менеджеров
  - `clients_*.json` - новые клиенты и адреса
  - `payments_*.json` - координаты оплат
- Сохраняет данные в центральную БД
- Используется для получения заказов с устройств

### 3. Полная синхронизация (выгрузка + загрузка)
- Выполняет оба действия последовательно
- Рекомендуется для регулярной синхронизации

## Настройка WebDAV

Перед использованием синхронизации необходимо настроить WebDAV:

1. Откройте вкладку **Настройки**
2. Перейдите на подвкладку **WebDAV**
3. Заполните поля:
   - **URL сервера:** `https://your-webdav-server.com/dav/`
   - **Логин:** имя пользователя
   - **Пароль:** пароль
4. Нажмите **Тест подключения** для проверки
5. Нажмите **Сохранить**

## Структура папок на WebDAV

```
/webdav/
  {PIN_менеджера}/
    incoming/          # Файлы для загрузки на устройство
      products.json    # Список товаров
      settings.json    # Настройки системы
    outgoing/          # Файлы от устройства
      orders_*.json    # Заказы
      clients_*.json   # Клиенты
      payments_*.json  # Координаты оплат
    processed/         # Обработанные файлы (архив)
```

## Лог синхронизации

Во время синхронизации в окне диалога отображается детальный лог:
- Подключение к WebDAV серверу
- Генерация файлов
- Загрузка/скачивание файлов
- Обработка данных
- Статистика: количество обработанных файлов, ошибки

## Работа без WebDAV

Если WebDAV не настроен, синхронизация будет работать только с локальными файлами в папке `data/webdav/`.

Это полезно для:
- Тестирования
- Работы в локальной сети без интернета
- Ручного переноса файлов через USB/email

## Автоматическая синхронизация

Помимо ручной синхронизации, система поддерживает автоматическую синхронизацию:
- Настраивается во вкладке **Настройки → Синхронизация**
- Интервал: от 1 до 60 минут
- Работает в фоновом режиме
- Можно включить/выключить

## Устранение проблем

### Ошибка подключения к WebDAV
- Проверьте URL сервера (должен начинаться с `http://` или `https://`)
- Проверьте логин и пароль
- Убедитесь, что сервер доступен (ping, браузер)
- Проверьте настройки файрвола

### Ошибка 401 (Unauthorized)
- Неверный логин или пароль
- Проверьте настройки WebDAV

### Ошибка 404 (Not Found)
- Неверный URL сервера
- Папка менеджера не существует на сервере

### Файлы не обрабатываются
- Проверьте формат JSON (должен быть валидным)
- Проверьте структуру данных (соответствие схеме)
- Посмотрите лог синхронизации для деталей ошибки

## Рекомендации

1. **Регулярная синхронизация:** Запускайте полную синхронизацию 2-3 раза в день
2. **Резервное копирование:** Делайте бэкап БД перед массовой загрузкой данных
3. **Мониторинг:** Проверяйте лог синхронизации на наличие ошибок
4. **Тестирование:** Сначала протестируйте на одном менеджере, затем масштабируйте
5. **Безопасность:** Используйте HTTPS для WebDAV, храните пароли в безопасности
