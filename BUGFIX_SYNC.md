# Исправление ошибки импорта заказов из WebDAV

## Проблема
При импорте заказов с мобильного устройства возникали ошибки:
1. ```
   UNIQUE constraint failed: clients.external_id
   This Session's transaction has been rolled back due to a previous exception during flush
   ```
2. ```
   UNIQUE constraint failed: addresses.external_id
   ```

## Причина
1. При создании клиента из заказа (метод `_get_or_create_client`) не устанавливался `external_id`
2. При последующем импорте файла `clients_*.json` пытался установить `external_id`, который уже существовал
3. При создании адреса не проверялся существующий `external_id`
4. После ошибки транзакция откатывалась, но сессия не очищалась, что приводило к повторным ошибкам

## Исправления в `src/sync/sync_manager.py`

### 1. Метод `_process_clients` (строка 392)
**Изменения:**
- Добавлен поиск клиента по `external_id` ИЛИ по телефону
- Добавлена проверка на конфликт `external_id` перед обновлением
- Добавлен `session.commit()` после каждого клиента
- Добавлен `session.rollback()` в блоке except для очистки сессии

### 2. Метод `_process_orders` (строка 167)
**Изменения:**
- Добавлен `session.commit()` после каждого заказа
- Добавлен `session.rollback()` в блоке except для очистки сессии

### 3. Метод `_process_payments` (строка 473)
**Изменения:**
- Добавлен `session.commit()` после каждой оплаты
- Добавлен `session.rollback()` в блоке except для очистки сессии

### 4. Метод `_get_or_create_address` (строка 300) ⭐ НОВОЕ
**Изменения:**
- Добавлен поиск адреса по `external_id` перед созданием
- Если адрес найден по `external_id`, обновляется его привязка к текущему клиенту
- Если не найден по `external_id`, ищется по `client_id` + `address_text`
- Предотвращает конфликты UNIQUE constraint для `addresses.external_id`

1. Запустите центральное приложение:
   ```bash
   cd E:\WORK_RUCHEEK\OrderManager\Central_office
   ./run.bat
   ```

2. Создайте заказ на мобильном устройстве с новым клиентом

3. Выполните синхронизацию (экспорт с мобильного)

4. В центральном приложении выполните импорт из WebDAV

5. Проверьте, что:
   - Заказ импортирован успешно
   - Клиент создан с правильным `external_id` (формат: `PIN-CLIENT_ID`)
   - Нет ошибок UNIQUE constraint
   - Если ошибка возникла для одного заказа, следующие заказы обрабатываются нормально

## Логика работы

### До исправления:
```
1. Импорт orders_*.json → создается клиент БЕЗ external_id
2. Импорт orders_*.json → создается адрес с external_id="ADDR-1"
3. Повторный импорт → попытка создать адрес с тем же external_id
4. ОШИБКА: external_id уже существует
5. Импорт clients_*.json → попытка установить external_id клиента
6. ОШИБКА: external_id уже существует (от другого менеджера)
7. Транзакция откатывается, но сессия не очищается
8. Все последующие операции падают с ошибкой "transaction has been rolled back"
```

### После исправления:
```
1. Импорт orders_*.json → создается клиент БЕЗ external_id
2. Импорт orders_*.json → поиск адреса по external_id
   - Если найден → обновляется привязка к текущему клиенту
   - Если не найден → создается новый адрес
3. Commit после каждого заказа
4. Импорт clients_*.json → поиск по телефону ИЛИ external_id
5. Клиент найден по телефону → обновляется external_id (если не конфликтует)
6. Commit после каждого клиента
7. Если ошибка → rollback и продолжение со следующим клиентом
```
