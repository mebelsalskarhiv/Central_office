# Вкладка "Отчеты" - Документация

## Описание
Новая вкладка "Отчеты" предоставляет два типа отчетов для эффективной работы кассира и курьера.

## Расположение
Вкладка находится между "Заказы" и "Клиенты" в главном окне приложения.

## Функциональность

### 1. Отчет для кассира (сборка товаров)

**Назначение:** Сводная таблица всех товаров, которые нужно собрать на выбранную дату.

**Формат:**
```
Товар          | Количество | Единица | Заказов
------------------------------------------------
Вода 1         | 13         | шт      | 3
Хлеб белый     | 5          | шт      | 2
Молоко 3.2%    | 8          | шт      | 4
```

**Возможности:**
- Фильтрация по дате
- Фильтрация по статусу заказа (Новые, В сборке, В доставке, Доставлено)
- Сортировка по любой колонке
- Печать отчета
- Экспорт в текстовый файл
- Итоговая статистика: всего товаров, наименований, заказов

**Пример использования:**
1. Выберите дату доставки
2. Выберите статус "Новые" или "В сборке"
3. Нажмите "Обновить"
4. Распечатайте отчет для кассира
5. Кассир собирает товары по списку

### 2. Отчет для курьера (доставка по адресам)

**Назначение:** Детальный список заказов, сгруппированных по клиентам и адресам.

**Формат:**
```
================================================================================
ОТЧЕТ ДЛЯ КУРЬЕРА - 03.05.2026
================================================================================

КЛИЕНТ: Дмитрий (+79281527177)
--------------------------------------------------------------------------------
  Адрес: Сальск, Шкирятова 117

    Заказ №ORD-001
    Время: 10:00-12:00
    Сумма: 1250.00 руб.
    Оплата: CASH
    Статус: UNPAID

    Состав заказа:
      - Вода 1: 3 шт x 50.00 = 150.00 руб.
      - Хлеб белый: 2 шт x 45.00 = 90.00 руб.
      - Молоко 3.2%: 5 шт x 80.00 = 400.00 руб.

    Комментарий: Позвонить за 30 минут

  Адрес: Сальск, Ленина 25

    Заказ №ORD-002
    Время: 14:00-16:00
    Сумма: 850.00 руб.
    Оплата: CARD
    Статус: PAID

    Состав заказа:
      - Вода 1: 7 шт x 50.00 = 350.00 руб.
      - Сок апельсиновый: 3 шт x 120.00 = 360.00 руб.

================================================================================
ИТОГО ЗАКАЗОВ: 2
================================================================================
```

**Возможности:**
- Группировка по клиентам
- Группировка по адресам внутри клиента
- Детальная информация по каждому заказу:
  - Номер заказа
  - Время доставки
  - Сумма
  - Способ оплаты
  - Статус оплаты
  - Состав заказа с ценами
  - Комментарии
- Печать отчета
- Экспорт в текстовый файл

**Пример использования:**
1. Выберите дату доставки
2. Выберите статус "В доставке"
3. Нажмите "Обновить"
4. Распечатайте отчет для курьера
5. Курьер развозит заказы по списку

## Фильтры

### Дата
Выбор даты доставки заказов. По умолчанию - текущая дата.

### Статус
- **Все** - все заказы независимо от статуса
- **Новые** - только что созданные заказы
- **В сборке** - заказы, которые собираются на складе
- **В доставке** - заказы в процессе доставки
- **Доставлено** - уже доставленные заказы

## Технические детали

### Файл
`src/gui/tabs/reports_tab.py`

### Зависимости
- PyQt6 (UI)
- SQLAlchemy (запросы к БД)
- collections.defaultdict (группировка данных)

### Запросы к БД
```python
# Получение заказов за дату с фильтром по статусу
query = session.query(Order).filter(
    Order.delivery_date >= date_from,
    Order.delivery_date <= date_to
)
if status_filter:
    query = query.filter(Order.status == status_filter)
orders = query.all()
```

### Группировка данных

**Для кассира:**
```python
product_stats = defaultdict(lambda: {"quantity": 0, "orders": set()})
for order in orders:
    for item in order.items:
        product_stats[item.product_name]["quantity"] += item.quantity
        product_stats[item.product_name]["orders"].add(order.id)
```

**Для курьера:**
```python
client_orders = defaultdict(lambda: defaultdict(list))
for order in orders:
    client_key = f"{client.name} ({client.phone})"
    client_orders[client_key][order.address_text].append(order)
```

## Примечания

- Отчеты генерируются в реальном времени из БД
- Данные обновляются при изменении даты или статуса
- Печать использует стандартный диалог печати Qt
- Экспорт сохраняет файлы в кодировке UTF-8
- Номер телефона - уникальный ключ клиента
- Один клиент может иметь несколько адресов
- Один адрес может иметь несколько заказов
